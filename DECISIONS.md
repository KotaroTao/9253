## DECISION-001: Docker不使用、GitHub + PM2で直接デプロイ

- **日付**: 2026-02-08
- **決定**: Docker Compose を使用せず、GitHub + PM2 による直接デプロイを採用
- **理由**: 開発者がGitHubベースのワークフローに慣れている。初期〜数千院規模ではDockerのオーバーヘッドは不要
- **デメリット（許容済み）**:
  - 開発PCとVPSの環境差異リスク → `.node-version` で管理
  - VPS初期セットアップが手動 → 1回きりの作業
  - 将来クラウド移行時にDocker化が必要 → その時点で対応
- **代替案**: Docker Compose（却下）

## DECISION-002: ポート3100を使用

- **日付**: 2026-02-08
- **決定**: Next.js開発サーバー・本番サーバーをlocalhost:3100で起動
- **理由**: localhost:3000〜3010は既存サービスで使用中
- **影響**: next.config.ts、PM2設定、Nginx設定で3100を指定する必要あり

## DECISION-003: 独自ドメインのクラウドSaaSとして提供

- **日付**: 2026-02-08
- **決定**: 独自ドメインを取得し、Web上のクラウドサービスとして提供

## DECISION-004: Googleポリシー完全準拠（レビューゲーティング禁止）

- **日付**: 2026-02-08
- **決定**: 競合の多くが採用しているレビューゲーティング（高評価のみGoogleに誘導）を一切行わない
- **理由**:
  - Googleポリシーの明確な違反であり、全レビュー削除・プロフィール停止のリスク
  - 2025年以降、GoogleのAI検出が強化されている
  - 法的リスク（FTC: 1件$51,744の罰金）
  - 「完全準拠」を差別化ポイントとして営業に活用
- **トレードオフ**: 口コミ数増加が競合より遅い可能性 → 削除支援機能とAI返信で補完

## DECISION-005: 景品表示法完全遵守（インセンティブ機能なし）

- **日付**: 2026-02-08
- **決定**: 口コミ投稿の見返りとしての割引・特典機能は実装しない
- **理由**:
  - 2025年3月、医療法人スマイルスクエアが口コミの見返りに治療費割引を行い措置命令
  - 課徴金（売上の3%）、2年以下の拘禁刑または300万円以下の罰金のリスク
- **代わりのアプローチ**: アンケートの「ついで」に全員に口コミ導線を提供（インセンティブなし）

## DECISION-006: 技術スタック選定

- **日付**: 2026-02-08
- **決定**: Next.js 15 + TypeScript + tRPC + Prisma + PostgreSQL 16 + Redis
- **理由**:
  - フルスタックTypeScriptで型安全な開発
  - Next.js App Routerで患者用ページ（SSR/SEO対応）と管理画面を1プロジェクトで管理
  - tRPCでAPI通信の型安全を確保
  - PrismaでスキーマファーストなDB管理
  - PostgreSQLは要件指定
- **UI**: Tailwind CSS + shadcn/ui（高速開発）
- **認証**: NextAuth.js v5
- **AI**: OpenAI GPT-4o API
