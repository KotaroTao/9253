## 1. エグゼクティブサマリー

### ビジョン
医療機関（歯科・クリニック・病院）特化のGoogle口コミ対策クラウドSaaS。Googleポリシー完全準拠・法令完全遵守を設計思想の根幹に置き、「口コミは、正しく増やす。」をコンセプトに業界標準ツールを目指す。

### 最重要原則（批判的レビュー反映）

> **コードを1行も書く前に、30院に電話してPMFを検証する。**
>
> 全ての計画は仮説であり、実際の顧客の声で検証・修正する。

### 4つの差別化ポイント
1. **口コミ削除支援の統合**（業界唯一）- 既存の削除解説ページ・削除判定ツールとの連携
2. **Googleポリシー完全準拠** - レビューゲーティング禁止を遵守した設計
3. **透明な価格** - 月額4,980円〜を公開（競合は料金非公開で推定3-5万円）
4. **3,000院の既存リスト** - 即座にアプローチ可能な営業基盤

### 既存資産
- Google口コミ削除解説ページ: https://function-t.com/delete_2025.html
- 口コミ削除判定ツール（ChatGPT GPTs）: https://chatgpt.com/g/g-68849d013b30819184bbc852185546a7
- 約3,000歯科医院のリスト

## 2. 市場分析と現実的な市場規模

### 2.1 市場推定（保守的見通し）

| 区分 | 施設数 | 根拠 |
|------|--------|------|
| 歯科医院 | 約68,000院 | 厚労省 医療施設動態調査 |
| 診療所（クリニック） | 約104,000院 | 同上 |
| 病院（20床以上） | 約8,000施設 | 同上 |
| **合計** | **約180,000施設** | |

| 市場区分 | 推定規模 | 算出根拠 |
|----------|---------|---------|
| **TAM**（全潜在市場） | 180,000施設 | 全医療施設 |
| **SAM**（対応可能市場） | 15,000施設 | GBP運用+口コミ対策予算あり（保守的8%） |
| **SOM**（初期獲得可能市場） | 3,000施設 | 既存リスト+初期マーケティング到達範囲 |

**批判的レビュー反映**: 当初のSAM 43,000施設は楽観的すぎる。ITリテラシーが低い個人開業医が多く、月額1万円のSaaS導入への抵抗が大きい。保守的に15,000施設と見積もる。

### 2.2 「シェア70%」の再定義

- 現状、口コミ対策ツールを導入している医療機関は推定5,000〜8,000施設
- **短期目標（36ヶ月）**: 口コミ対策ツール導入済み医療機関の中でのシェア70% = 3,500〜5,600施設
- **長期目標（60ヶ月）**: 市場自体を拡大し、新規開拓も含めて10,000施設以上

### 2.3 最大の敵は「競合」ではなく「無関心」

**批判的レビュー反映**: 日本の歯科医院の大半は口コミ対策を行っていない。市場の教育コストが最も大きな投資項目。口コミ対策の必要性を理解していない院長を説得するコンテンツ（セミナー、ホワイトペーパー、成功事例動画）にマーケティング予算の50%以上を充てる。

### 3.2 競合の反撃シナリオと対策

**批判的レビュー反映**: 競合は黙っていない。5つの反撃シナリオを事前に想定する。

| シナリオ | 発生確率 | 対策 |
|----------|---------|------|
| **口コミ365がポリシー準拠モードを追加** | 高 | 機能模倣は数週間で可能だが、既存顧客に「今まで違反していた」と説明するリスクを負う。「最初からポリシー準拠」というブランドイメージは追随困難 |
| **口コミ365が医療特化テンプレートを追加** | 高 | テンプレートは模倣容易。差別化の軸を「機能」ではなく「医療業界の専門性と信頼（法務監修、業界人脈）」に置く |
| **競合が価格をダンピング** | 中 | 当社の月額4,980円〜はそもそも低価格。これ以上の値下げ競争には付き合わない。機能価値で勝負 |
| **口コミ365が口コミ削除支援を追加** | 低 | 削除支援のノウハウ蓄積には時間が必要。判例データベースの先行優位を維持 |
| **ヒーローイノベーションが口コミ対策を強化** | 中 | 既に医療特化で競合。価格透明性と削除支援機能で差別化 |
| **大手IT企業が参入** | 低 | 医療特化のニッチ市場は大手には小さすぎる。参入してもバーティカル知識で優位 |

### 3.3 持続的な差別化の軸

**批判的レビュー反映**: 機能は模倣される。本当のモートは「ブランドと信頼」。

1. **専門性**: 医療法・景品表示法の法務知識（弁護士顧問契約）
2. **データ蓄積**: 口コミ削除判例データベース（実運用でしか蓄積不可能）
3. **業界人脈**: 歯科経営コンサル・歯科医師会との連携
4. **顧客満足度**: 最初の100院で圧倒的なNPSを達成し、口コミで拡大
5. **スイッチングコスト**: 蓄積データ（口コミ分析履歴、NPS推移、MEO順位履歴）の価値

## 4. 法令・ポリシー準拠設計

### 4.1 Googleポリシー完全準拠

```
【患者フロー（全員に等しく口コミを依頼）】

受診完了 → QRコード/SMS → アンケートページ
                                    ↓
                          全員にアンケート回答依頼
                                    ↓
                        ┌─ アンケート結果を院内で活用（改善用）
                        │
                        └─ 全員にGoogle口コミ投稿を依頼（評価によらず）
                                    ↓
                          口コミ下書きAI支援（任意利用）
                                    ↓
                          患者が自分の言葉で確認・修正して投稿
```

**設計原則**: 高評価者のみをGoogleに誘導する仕組みは一切設けない。

### 4.2 景品表示法・ステマ規制対応

- **インセンティブ機能を実装しない**: 口コミの見返りの割引・特典は一切なし
- **2025年3月スマイルスクエア措置命令事例を教訓**: 口コミ投稿で治療費割引→消費者庁から措置命令
- **罰則**: 課徴金（売上の3%）、2年以下の拘禁刑または300万円以下の罰金

### 4.3 法的リスクへの対応（批判的レビュー反映）

**重要**: 以下は開発着手前に弁護士と確認すべき事項。

| リスク | 内容 | 対策 |
|--------|------|------|
| **口コミ削除支援の非弁行為リスク** | 弁護士法72条違反の可能性。「支援」「ガイド」でも実質的に代行と見なされうる | **開発前に弁護士の法的意見書を取得**。削除申請自体は医院が自ら行う設計。ツールはあくまで情報提供・テンプレート提供 |
| **AI下書きのステマリスク** | 事業者提供ツールで作成された口コミがステマ認定されうる | AI下書きの30%以上を患者が修正しないと投稿ボタンを有効にしない。「あなたの言葉で書き直してください」を必須ステップに |
| **個人情報保護（要配慮個人情報）** | アンケートが病歴関連情報に該当しうる | アンケート開始前に個人情報取得同意画面を必須表示。プライバシーポリシーは弁護士に作成依頼 |
| **SMS配信の法的制約** | 特定電子メール法によるオプトイン同意義務 | 医院側がオプトイン同意を取得済みであることの確認チェックボックスを必須化 |
| **「準拠証明書」の法的位置づけ** | 自社発行の「証明書」は誤認リスク | 「準拠証明書」→「弁護士監修 設計思想説明書」に変更 |
| **AI返信の個人情報漏洩** | 返信に患者の診療内容を推測する記述が含まれるリスク | 個人情報フィルター（患者名、診療内容、病名等を自動検出・除去）を実装 |

### 4.4 開発着手前の必須アクション

1. 弁護士と顧問契約を締結
2. 口コミ削除支援機能の合法性について書面での法的意見書を取得
3. プライバシーポリシー・利用規約の作成
4. アンケートにおける個人情報取得同意フローの法的チェック

### 6.3 技術的リスクと対策（批判的レビュー反映）

| リスク | 影響 | 対策 |
|--------|------|------|
| **GBP APIの申請が通らない** | 口コミ取得・返信投稿不可 | **MVP はGBP API非依存で設計**。申請は並行して進める。未承認の場合は手動インポート機能で代替 |
| **GBP APIのレート制限** | 5,000院で15分ポーリングは不可能 | 医院ごとにアダプティブ頻度制御（アクティブ:15分、非アクティブ:1時間） |
| **OpenAI APIのコスト増大** | 粗利圧迫 | GPT-4o-miniをデフォルト（コスト1/10）、プロンプトキャッシュ（Redis）、月次コスト上限設定 |
| **OpenAI APIのハルシネーション** | 医療情報の誤りが含まれるリスク | 医療用語NGワードフィルター、個人情報自動除去、院長の確認ステップ必須 |
| **VPS単一障害点** | 全サービス停止 | バックアップはVPS外に保管（S3互換）。1,000院超でクラウド移行を検討 |
| **PostgreSQL RLS設定ミス** | テナント間データ漏洩 | E2Eテストでテナント分離を自動検証。MVP段階ではアプリケーションレベルの分離を先行 |
| **Google Maps Platform料金改定** | MEO計測コスト増 | API使用量月次モニタリング、代替手段（自社クローラー）の準備 |
| **円安によるAPI費用増** | 運用コスト膨張 | 粗利率70%以上を維持する価格設計、年次価格改定条項 |

→ この結果次第で、計画のすべてを見直す

### 8.2 GTM戦略（保守的CVR前提）

**批判的レビュー反映**: CVR 10%は楽観的。リストの質次第で0.5-5%が現実的。3%で計画。

#### Phase 1: 最初の100院（Month 1-8）

1. **リストのスコアリング**: 3,000院をA/B/Cランクに分類
   - Aランク（800院）: Google口コミ10件以上、平均3.5以下 → 「痛み」が大きい
   - Bランク（1,200院）: 口コミ5-10件、対策未着手
   - Cランク（1,000院）: 口コミ5件未満
2. **ハイタッチ営業**: Aランクに創業者自らパーソナライズドメール+電話
   - 「{医院名}様の口コミを分析し、無料レポートを作成しました」
   - CVR 3% = 24院（Aランク800院×3%）
3. **無料トライアル**: 14日間全機能無料
4. **最初の10院は無料で使ってもらい、利用データを収集**
   - アンケート回答率、口コミ投稿率、AI下書き利用率
5. **目標**: 50院（有料） + 10院（無料パイロット）

#### Phase 2: 市場拡大（Month 9-24）

1. SEO/コンテンツマーケティング: 月8記事
2. Google広告: CPA 15,000円以内
3. 代理店チャネル（歯科ディーラー10社、Web制作会社20社）

## 9. 批判的吟味と改善策

### 9.1 批判的レビュワーが指摘した最重大リスクTOP 5

| # | リスク | 深刻度 | 対策 |
|---|--------|--------|------|
| 1 | **PMF未検証** — 医院が本当に口コミ対策ツールに金を払うか未確認 | 致命的 | **開発前に30院ヒアリング + 10院無料パイロット** |
| 2 | **実行リソース不足** — 1-2人で8週間MVPは不可能 | 致命的 | **MVP機能を大幅削減、スケジュールを3倍に** |
| 3 | **GBP API未承認** — コア機能がAPI前提なのに申請すらしていない | 高 | **MVPをAPI非依存で設計 + 今すぐ申請** |
| 4 | **口コミ削除支援の非弁行為リスク** — 弁護士法72条違反の可能性 | 高 | **開発前に弁護士の法的意見書取得** |
| 5 | **資金計画の欠如** — 36ヶ月40名体制の資金見通しなし | 高 | **悲観シナリオで月次キャッシュフロー計画作成** |

### 9.2 プラン修正の要約

| 項目 | 当初案 | 修正後 |
|------|--------|--------|
| MVP開発期間 | 8週間 | 12-16週間 |
| MVP機能数 | 10機能 | 5機能（アンケート+QR+口コミ導線+管理画面+認証） |
| 6ヶ月後導入目標 | 300院 | 50院 |
| 12ヶ月後MRR | 900万円 | 240万円（基本シナリオ） |
| 最低価格プラン | 月額9,800円 | 月額4,980円 |
| SLA | 99.9% | ベストエフォート（クラウド移行後にSLA提供） |
| 36ヶ月後体制 | 40名 | 5-8名 |
| 資金計画 | なし | 悲観シナリオで策定 |
| PMF検証 | なし | **開発前に30院ヒアリング必須** |

### 9.3 口コミ対策「だけ」ではない価値提案

**批判的レビュー反映**: 口コミは結果。原因（患者体験の改善）にアプローチするツールとして位置づけ直す。

> **「患者満足度を見える化し、口コミと集患を自然に増やすツール」**

- アンケートで患者満足度を測定（NPS）
- 低評価の原因を分析し改善提案
- 結果として口コミが自然に増える
- 口コミ管理・返信・削除支援も一元化
