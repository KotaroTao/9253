## 1. エグゼクティブサマリー

### ビジョン
医療機関（歯科・クリニック・病院）に特化した、Googleポリシー完全準拠・法令完全遵守の口コミ対策Webアプリケーションを開発する。既存の約3,000歯科医院リストを初期ターゲットとし、業界シェア70%超を目指す。

### 競合との決定的な差別化ポイント
1. **医療機関完全特化** - 汎用ツールではなく医療業界の法規制（医療法・景品表示法）に完全対応
2. **口コミ削除支援の統合** - 既存の削除解説ページ・削除判定ツールとの連携（業界唯一）
3. **Googleポリシー完全準拠** - レビューゲーティング禁止を遵守した設計
4. **3,000院の既存リスト** - 即座にアプローチ可能な営業基盤

### 2.2 競合の致命的弱点

#### 口コミ365（最大競合）
- **レビューゲーティングの疑い**: NGワード設定で低評価を排除する機能はGoogleポリシー違反の可能性
- **医療特化なし**: 医療法・薬機法への対応が不十分
- **価格不透明**: 料金非公開で営業依存の販売モデル
- **口コミ削除機能なし**: ネガティブ口コミへの根本的対策がない

#### 口コミPLUS
- **明確なレビューゲーティング**: 「高評価のみGoogleに誘導、低評価はツール内保存」はGoogleポリシーの明確な違反
- Googleからの制裁リスク（全レビュー削除・プロフィール停止）を顧客に転嫁

#### その他競合共通の弱点
- 医療機関特有のコンプライアンス（患者プライバシー、医療広告ガイドライン）への対応が不十分
- 口コミ削除申請のサポートがない
- 歯科・クリニック向けのアンケート設問テンプレートがない

## 3. 法令・ポリシー準拠設計

### 3.1 Googleポリシー完全準拠

#### 遵守すべき要件
1. **レビューゲーティング禁止**: 全ての患者に等しく口コミを依頼する仕組み
2. **偽レビュー禁止**: AI生成テキストのそのままの投稿禁止（あくまで下書き支援）
3. **インセンティブ付き口コミ禁止**: 口コミと引き換えの割引・特典の禁止

#### 本製品の準拠設計

```
【患者フロー（Googleポリシー完全準拠）】

受診完了 → QRコード/SMS → アンケートページ
                                    ↓
                          全員にアンケート回答依頼
                                    ↓
                        ┌─ アンケート結果を院内で活用（改善用）
                        │
                        └─ 全員にGoogle口コミ投稿を依頼（評価によらず）
                                    ↓
                          口コミ下書きAI支援（任意利用）
                                    ↓
                          患者が自分の言葉で投稿
```

**重要**: 高評価者のみをGoogleに誘導する仕組みは一切設けない。
全患者に対して等しくGoogle口コミページへの導線を提供する。

### 3.2 景品表示法・ステルスマーケティング規制

#### 2023年10月施行・2025年以降の強化ポイント
- 口コミの見返りとしての割引・特典提供 → **措置命令対象**（2025年3月：スマイルスクエア事例）
- 口コミ代行・偽レビュー → **景表法違反＋Googleポリシー違反**
- 課徴金: 売上の3%、違反時: 2年以下の拘禁刑または300万円以下の罰金

#### 本製品の対策
1. **インセンティブ機能を実装しない**: 口コミ投稿に対する報酬・割引機能は設けない
2. **AI下書き支援の明確な位置づけ**: AIは文章作成を補助するのみ、患者自身が内容を確認・修正して投稿
3. **コンプライアンスチェッカー**: 管理画面内で景品表示法違反リスクをアラート表示
4. **監査ログ**: 全操作の記録保持（有事の証拠として利用可能）

### 3.3 医療広告ガイドライン

- 体験談の広告利用制限への対応
- 患者プライバシー保護（個人情報保護法）
- 医療法における広告規制

### 3.4 法令準拠の証明・差別化

- **「Googleポリシー準拠証明書」の発行**: 導入医院に対して準拠設計であることを文書化
- **法令準拠レポート**: 定期的にコンプライアンス状況を可視化
- **競合比較資料**: レビューゲーティングの危険性を解説する営業資料として活用

## 4. 製品仕様

### 4.1 コア機能

#### A. 患者アンケート・口コミ収集システム

| 機能 | 詳細 |
|------|------|
| **QRコード生成** | 医院ごとのカスタムQRコード。受付・待合室・診察券に掲載 |
| **SMSリンク送信** | 受診後に自動でアンケートリンクをSMS送信 |
| **LINEミニアプリ連携** | LINE公式アカウントからアンケート導線 |
| **アンケートビルダー** | 診療科目別テンプレート（歯科・内科・皮膚科・整形外科等） |
| **NPS計測** | Net Promoter Scoreで患者満足度を定量化 |
| **全員口コミ導線** | アンケート回答後、全員にGoogle口コミページリンクを表示 |

**歯科専用アンケートテンプレート例:**
- 治療の痛みについて（1-5段階）
- 待ち時間について（1-5段階）
- スタッフの対応について（1-5段階）
- 院内の清潔さについて（1-5段階）
- 治療の説明のわかりやすさについて（1-5段階）
- 自由記述（任意）

#### B. AI口コミ下書き支援

| 機能 | 詳細 |
|------|------|
| **アンケート回答からの下書き生成** | 選択式回答を基にAIが自然な口コミ文を生成 |
| **患者編集画面** | 下書きを患者が自由に修正可能（必須ステップ） |
| **医療用語最適化** | 歯科・医科の専門用語を自然な患者目線の表現に変換 |
| **多言語対応** | 日本語・英語・中国語・韓国語 |
| **NGワードチェック** | 医療広告ガイドラインに抵触する表現を自動検出・警告 |

**重要設計原則**: AIが生成した文章をそのまま投稿させない。患者が確認・修正するステップを必須とする。

#### C. 口コミ返信AI

| 機能 | 詳細 |
|------|------|
| **自動返信下書き生成** | 新着口コミに対してAIが返信案を自動生成 |
| **医療機関向けトーン** | 丁寧・専門的・共感的なトーンをプリセット |
| **返信テンプレート** | 診療科目・内容別の返信テンプレートライブラリ |
| **HIPAA/個人情報配慮** | 返信内容に患者の個人情報・診療内容を含めない自動チェック |
| **ワンクリック投稿** | Google Business Profile APIを通じて直接返信 |

#### D. 口コミ削除支援（業界唯一の機能）

| 機能 | 詳細 |
|------|------|
| **削除判定AI** | 既存の削除判定ツールを統合。Googleポリシー違反を自動判定 |
| **削除申請ガイド** | ステップバイステップの削除申請手順 |
| **削除申請テンプレート** | ポリシー違反カテゴリ別の申請文テンプレート |
| **削除追跡** | 申請状態のトラッキング（申請中・承認・却下） |
| **法的対応ガイド** | 名誉毀損等の法的措置が必要な場合の弁護士連携案内 |
| **削除解説コンテンツ** | 既存の `function-t.com/delete_2025.html` を統合 |

#### E. MEO順位計測・分析

| 機能 | 詳細 |
|------|------|
| **キーワード順位追跡** | 「地域名 + 診療科」のGoogleマップ順位を日次計測 |
| **競合比較** | 同エリア競合医院との口コミ数・評価比較 |
| **口コミ分析ダッシュボード** | 口コミ数推移・評価推移・感情分析・キーワード分析 |
| **月次レポート自動生成** | PDFレポートを自動生成して院長にメール送信 |
| **GBP投稿連携** | Googleビジネスプロフィールへの投稿管理 |

#### F. 患者満足度改善サイクル

| 機能 | 詳細 |
|------|------|
| **低評価アラート** | 低評価アンケートを即座にSlack/メール通知 |
| **改善提案AI** | 低評価の傾向分析から具体的な改善策を提案 |
| **改善トラッキング** | 改善施策の実施と効果測定を一元管理 |
| **ベンチマーク** | 同規模・同地域の医院との比較データ |

### 5.3 データベース設計（主要テーブル）

```sql
-- 医院（テナント）
CREATE TABLE clinics (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(255) NOT NULL,
    google_place_id VARCHAR(255),
    gbp_account_id VARCHAR(255),
    prefecture VARCHAR(50),
    city VARCHAR(100),
    address TEXT,
    phone VARCHAR(20),
    specialty VARCHAR(50), -- dental, internal, dermatology, etc.
    plan_type VARCHAR(20) DEFAULT 'trial',
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- ユーザー（医院スタッフ）
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    clinic_id UUID REFERENCES clinics(id),
    email VARCHAR(255) UNIQUE NOT NULL,
    name VARCHAR(100),
    role VARCHAR(20) DEFAULT 'staff', -- owner, admin, staff
    password_hash TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT NOW()
);

-- アンケート定義
CREATE TABLE surveys (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    clinic_id UUID REFERENCES clinics(id),
    title VARCHAR(255),
    questions JSONB NOT NULL, -- 設問定義
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMP DEFAULT NOW()
);

-- アンケート回答
CREATE TABLE survey_responses (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    survey_id UUID REFERENCES surveys(id),
    clinic_id UUID REFERENCES clinics(id),
    answers JSONB NOT NULL,
    nps_score INTEGER,
    submitted_at TIMESTAMP DEFAULT NOW()
);

-- Google口コミ
CREATE TABLE google_reviews (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    clinic_id UUID REFERENCES clinics(id),
    google_review_id VARCHAR(255) UNIQUE,
    author_name VARCHAR(255),
    rating INTEGER NOT NULL,
    text TEXT,
    reply_text TEXT,
    reply_status VARCHAR(20) DEFAULT 'pending', -- pending, drafted, replied
    ai_reply_draft TEXT,
    sentiment VARCHAR(20), -- positive, neutral, negative
    keywords JSONB,
    published_at TIMESTAMP,
    fetched_at TIMESTAMP DEFAULT NOW()
);

-- 口コミ削除申請
CREATE TABLE review_deletion_requests (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    clinic_id UUID REFERENCES clinics(id),
    google_review_id VARCHAR(255),
    violation_type VARCHAR(50), -- spam, fake, offensive, conflict_of_interest, etc.
    ai_assessment JSONB, -- AI判定結果
    status VARCHAR(20) DEFAULT 'draft', -- draft, submitted, approved, rejected
    notes TEXT,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- MEO順位記録
CREATE TABLE meo_rankings (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    clinic_id UUID REFERENCES clinics(id),
    keyword VARCHAR(255) NOT NULL,
    rank_position INTEGER,
    recorded_at DATE DEFAULT CURRENT_DATE,
    UNIQUE(clinic_id, keyword, recorded_at)
);

-- GBP投稿
CREATE TABLE gbp_posts (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    clinic_id UUID REFERENCES clinics(id),
    content TEXT NOT NULL,
    image_url TEXT,
    post_type VARCHAR(20), -- update, event, offer
    status VARCHAR(20) DEFAULT 'draft', -- draft, scheduled, published
    scheduled_at TIMESTAMP,
    published_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 監査ログ
CREATE TABLE audit_logs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    clinic_id UUID REFERENCES clinics(id),
    user_id UUID REFERENCES users(id),
    action VARCHAR(100) NOT NULL,
    details JSONB,
    ip_address INET,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 月次レポート
CREATE TABLE monthly_reports (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    clinic_id UUID REFERENCES clinics(id),
    report_month DATE NOT NULL,
    data JSONB NOT NULL,
    pdf_url TEXT,
    generated_at TIMESTAMP DEFAULT NOW()
);
```

## 7. 批判的吟味と差別化戦略

### 7.1 本プランの潜在リスクと対策

#### リスク1: Googleポリシー完全準拠 → 口コミ数増加が競合より遅い
**分析**: レビューゲーティングを行わないため、ネガティブ口コミも含まれる。
**対策**:
- 口コミ「数」だけでなく「質」と「返信の充実度」がMEOに効果的であることをデータで証明
- 低評価口コミへの迅速・丁寧な返信がむしろ信頼性を高めることを訴求
- 口コミ削除支援機能でポリシー違反レビューを適切に排除（合法的アプローチ）
- **実績データ**: 競合のレビューゲーティング利用院がGoogleから制裁を受けた事例を営業資料化

#### リスク2: 口コミ365が既に3,000店突破の先行優位
**分析**: 先行者利益は大きいが、汎用ツールの弱点を突ける。
**対策**:
- 医療機関「完全特化」で汎用ツールにはない機能（口コミ削除支援、医療法対応、診療科別テンプレート）を訴求
- 既存3,000院リストへの直接アプローチ（競合は代理店経由の間接販売が多い）
- 価格透明性で差別化（口コミ365は料金非公開）
- 乗り換えキャンペーンの実施

#### リスク3: AI口コミ下書き機能のグレーゾーン
**分析**: AIが口コミ文を生成すること自体への法的・倫理的懸念。
**対策**:
- AIはあくまで「下書き支援」。患者が内容を確認・修正して自分の意思で投稿する設計
- 「AI生成のままの投稿」を推奨しないUI設計（編集を促すステップを挟む）
- 利用規約に明記：「本ツールは患者自身の投稿を補助するものであり、虚偽の口コミの作成を目的としません」

#### リスク4: Google Business Profile API の制限・変更リスク
**分析**: GoogleがAPIを変更・制限する可能性。
**対策**:
- APIに依存しない基本機能の確保（アンケート・NPS計測はAPIなしで動作）
- スクレイピングに頼らない正規API利用
- 複数の口コミプラットフォーム対応で単一依存を回避

#### リスク5: エックスサーバーVPSの性能限界
**分析**: ユーザー数が拡大した場合のスケーラビリティ。
**対策**:
- Docker化により、将来的にクラウド（AWS/GCP）への移行が容易な設計
- Phase 2以降でクラウド移行を計画
- CDN活用でフロントエンドの負荷を分散

### 7.2 競合が追随できない差別化要素

1. **口コミ削除支援の統合**: 既存の削除判定ツール・解説コンテンツとの連携は同業他社が持たない資産
2. **3,000院リスト**: 初期アプローチ先を既に確保
3. **Googleポリシー完全準拠の証明**: 競合の多くがレビューゲーティングを行っている中、堂々と「完全準拠」を謳える
4. **医療法・景品表示法の専門知識**: function-t.comの既存コンテンツが信頼性の証拠
5. **価格透明性**: 競合が料金を隠す中、明確な価格を公開

### 7.3 Go-to-Market戦略

#### Phase 1: 既存リスト攻略（0-6ヶ月）
1. 3,000歯科院リストにメール+電話でアプローチ
2. 「Googleポリシー完全準拠」と「口コミ削除支援」を前面に打ち出す
3. 競合（口コミ365等）のレビューゲーティングリスクを教育するコンテンツマーケティング
4. 14日間無料トライアルで導入障壁を下げる

#### Phase 2: 拡大（6-18ヶ月）
1. 歯科以外の医療機関（内科、皮膚科、整形外科等）への展開
2. 歯科ディーラー・医療コンサルとの代理店契約
3. SEO/コンテンツマーケティング強化（「Google口コミ 歯科 対策」等）
4. 成功事例の蓄積と公開

#### Phase 3: 市場支配（18-36ヶ月）
1. 医療機関向け口コミ対策の第一想起ブランドを確立
2. 医療系学会・セミナーでの露出
3. API連携（電子カルテ・予約システム）で医院のワークフローに統合
4. 病院グループ・医療法人向けエンタープライズプラン
