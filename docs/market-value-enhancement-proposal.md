# MIERU PX 市場価値向上提案書（方針転換後）

## 前提: 方針転換の整理

### 転換前 → 転換後

| 項目 | 転換前 | 転換後 |
|---|---|---|
| 主軸 | 患者満足度向上 + 口コミ獲得 | **患者満足度向上に完全特化** |
| スタッフ評価 | 個別スタッフのスコア追跡 | **チーム全体の改善** |
| Google口コミ | 口コミへの誘導あり | **口コミ誘導なし**（月次推移の参考記録のみ） |
| 競合との比較軸 | 「コンプライアンス準拠の口コミツール」 | **「口コミツールではない。患者体験改善ツール」** |

### 転換後の現在の機能一覧

| 機能 | 状態 | 概要 |
|---|---|---|
| キオスクアンケート | 実装済 | タブレットで30秒、来院種別・診療種別・年代等を収集 |
| QRコードアンケート | 実装済 | ラミネートカードから患者自身のスマホで回答 |
| 満足度ダッシュボード | 実装済 | 全体スコア、月次推移、質問別スコア |
| 最近のフィードバック一覧 | 実装済 | 直近10件のスコア・属性・フリーテキスト表示 |
| 月次レポート | 実装済 | 来院数・売上入力 → 自費率・患者単価等を自動算出 |

| スタッフ管理 | 実装済 | 登録・有効/無効切替（個別評価は非表示） |
| キオスク日次目標 | 実装済 | 1日15件目標、連続日数ストリーク、マイルストーン |
| 歯の豆知識 | 実装済 | アンケート完了後に60種のtipsをランダム表示 |

## 核心的な問い: 「口コミ」を外したMIERU PXは何で戦うのか？

口コミツールを外したことで、競合（口コミPLUS、Hero Innovation等）との直接比較から離脱した。これは**弱みではなく、カテゴリ創出のチャンス**である。

日本市場に「患者体験改善プラットフォーム」は存在しない。MIERU PXは**新しいカテゴリの定義者**になれる。

ただし、現状の機能は「データを集めて表示する」段階にとどまっている。
**「データ → 気づき → 行動 → 改善」のサイクルを回す仕組み**が不足している。

ここを埋めることが市場価値を飛躍的に高めるカギになる。

---

## 提案: 市場価値を高める7つの施策

---

### 施策1: 「改善サイクル」機能 — データを行動に変える（最重要）

**現状の課題**: ダッシュボードにスコアとグラフは表示されるが、「だから何をすればいいのか」が分からない。院長はデータを見ても具体的なアクションに落とせない。

**提案**: 質問別スコアの変動を検知し、具体的な改善アクションカードを自動生成する。

#### 実装イメージ

```
┌─────────────────────────────────────────────┐
│  📋 今月の改善ポイント                         │
├─────────────────────────────────────────────┤
│                                             │
│  ▼ 待ち時間の満足度が先月比 -0.6pt            │
│    推奨アクション:                            │
│    ・予約間隔の見直し（現在の間隔+5分を検討）    │
│    ・待合室に所要時間の目安を掲示               │
│    ・待ち時間が長くなる場合の声かけルールを設定    │
│                                             │
│  ▲ 説明のわかりやすさ が先月比 +0.4pt          │
│    好調を維持:                                │
│    ・この取り組みをチームで共有しましょう         │
│                                             │
└─────────────────────────────────────────────┘
```

#### なぜ最重要か
- 「データを見せる」ツールは世にある。**「データから行動を導く」ツール**は日本の小規模クリニック向けに存在しない
- 院長の本音は「忙しくてデータを分析する時間がない。何をすべきか教えてほしい」
- 改善アクションのテンプレートは歯科経営コンサルの知見をベースに事前定義可能（AI不要でMVP実装可能）

#### 実装レベル

| レベル | 内容 | 実装コスト |
|---|---|---|
| Level 1 | 質問カテゴリ別にスコア低下を検知 → 事前定義したアクション候補を表示 | 低 |
| Level 2 | 患者属性（初診/再診、年代等）とクロスして「初診×待ち時間」のように絞り込んだ改善提案 | 中 |
| Level 3 | フリーテキストの頻出キーワードを抽出し、具体的な患者の声と紐づけた提案 | 高（AI活用） |

---

### 施策2: 患者属性クロス分析 — 「誰が不満か」を特定する

**現状の課題**: `patientAttributes`（来院種別・診療種別・年代・性別・主訴）はDBに保存されているが、ダッシュボードではフィルタリングに使われていない。

**提案**: 患者属性をフィルタとして使えるようにし、「どのセグメントに課題があるか」を一目で分かるようにする。

#### 発見できるインサイトの例

| フィルタ | 発見 | アクション |
|---|---|---|
| 初診 vs 再診 | 初診4.0 / 再診4.6 | 初診体験（受付案内、院内説明）に改善余地 |
| 治療 vs 定期検診 | 治療3.8 / 検診4.5 | 治療時の説明・痛みケアに課題 |
| 30代 vs 60代 | 30代3.9 / 60代4.4 | 30代向けの情報提供（デジタル予約等）が不足 |
| 主訴別 | 矯正相談4.1 / 痛み対応3.7 | 急患対応のプロセス改善が必要 |

#### 実装イメージ

```
┌─ フィルタ ──────────────────────────────────┐
│ 来院区分: [全て ▼]  診療内容: [全て ▼]        │
│ 年代:    [全て ▼]  期間:    [今月 ▼]         │
└──────────────────────────────────────────────┘

  全体: 4.3 → フィルタ適用後: 3.8（初診×治療）
  ⚠ このセグメントは全体平均を0.5pt下回っています
```

#### なぜ重要か
- 既にデータは取得済み（追加の患者負担ゼロ）
- 「全体スコア4.3で安心」→「初診の治療患者は3.8」という発見が院長の行動を変える
- 日本の競合で患者属性クロス分析を提供するツールはゼロ

---

### 施策3: 「改善の軌跡」レポート — 院長のモチベーションを維持する

**現状の課題**: 月次推移チャートはあるが、「何をした結果、何が変わったか」を記録する仕組みがない。改善活動は3ヶ月で形骸化しがちである。

**提案**: 院長が「今月やったこと」をメモとして記録でき、スコア推移と並べて表示する機能。

#### 実装イメージ

```
満足度推移グラフ
  4.5 ┤        ╭──○──────
  4.3 ┤    ╭───╯
  4.1 ┤────╯
  3.9 ┤
      └──1月──2月──3月──4月──5月──
           │       │
           │       └─「治療説明テンプレートを導入」
           └─「待合室に所要時間掲示を設置」

  改善メモ:
  ┌────────────────────────────────────────┐
  │ 3月: 待合室に所要時間掲示を設置           │
  │ 4月: 治療説明テンプレートを導入（結果: +0.4pt）│
  └────────────────────────────────────────┘
```

#### なぜ重要か
- 「アンケートを取り始めたが、結局何も変わらなかった」が最大の解約理由になる
- 改善活動の記録があると「やった→変わった」の因果が見え、継続のモチベーションになる
- 解約防止（チャーン抑制）に直結する
- 実装コストは非常に低い（テキストフィールド+タイムスタンプのみ）

---

### 施策4: フリーテキスト分析 — 「患者の声」をもっと活かす

**現状の課題**: フリーテキスト（最大500文字）は回答一覧に表示されるが、件数が増えると全部読むのは困難。傾向の把握ができない。

**提案**: フリーテキストをカテゴリ分類し、ワードクラウドや頻出テーマで可視化する。

#### 段階的実装

| 段階 | 内容 | 技術 |
|---|---|---|
| Phase 1 | キーワードマッチングによるタグ付け（「待ち時間」「説明」「痛み」「清潔」「受付」等） | 正規表現マッチング |
| Phase 2 | ポジティブ/ネガティブ自動分類 + 月別の傾向表示 | 辞書ベース感情分析 |
| Phase 3 | AI要約（月間フリーテキストの要点を3行で） | LLM API |

#### Phase 1の実装イメージ（低コスト）

```
┌─ 今月の患者の声（フリーテキスト分析） ────────┐
│                                             │
│  頻出テーマ:                                 │
│  ■■■■■■■■ 待ち時間 (12件)                   │
│  ■■■■■    説明     (8件)                    │
│  ■■■      受付対応  (5件)                    │
│  ■■       清潔さ   (3件)                    │
│                                             │
│  「待ち時間」の声（抜粋）:                     │
│  ・「予約したのに30分以上待った」               │
│  ・「待ち時間は長いが、診療は丁寧だった」        │
│  ・「前回より待ち時間が短くなっていた」          │
│                                             │
└──────────────────────────────────────────────┘
```

#### なぜ重要か
- スコアは「何点か」を示すが、フリーテキストは「なぜその点数か」を示す
- 院長が最も読みたいのはフリーテキストだが、量が増えると読めなくなる
- Phase 1（キーワードマッチング）なら外部AI不要、実装コスト低

### 施策5: 従業員満足度 × 患者満足度の相関可視化 — PX＋EXの一体化

**現状の機能**: 患者満足度（PX）と従業員満足度調査（EX）は別々のページに存在し、両者の関係が見えない。

**提案**: 同一ダッシュボード上でPXとEXを並列表示し、相関を可視化する。

#### 実装イメージ

```
┌─ 満足度の全体像 ─────────────────────────────┐
│                                             │
│  患者満足度 (PX)          従業員満足度 (EX)    │
│  ━━━━━━━━━━━━━          ━━━━━━━━━━━━━━     │
│    4.3 / 5.0               3.9 / 5.0        │
│    ▲ +0.2 (先月比)         ▼ -0.1 (前回比)   │
│                                             │
│  ┌──────────────────────────────────────┐   │
│  │ 相関メッセージ:                        │   │
│  │ 「従業員の"やりがい"スコアが高い月は    │   │
│  │  患者満足度も高い傾向があります」       │   │
│  └──────────────────────────────────────┘   │
│                                             │
│  💡 EXの「待遇・評価」カテゴリが低下傾向。    │
│     スタッフケアが患者体験にも影響する可能性。  │
│                                             │
└──────────────────────────────────────────────┘
```

#### なぜ重要か
- **「スタッフが幸せでなければ患者も幸せにできない」** — これは医療経営の定説だが、数値で示せるツールが日本にない
- PX単体のツールは「アンケートツール」、PX+EXを統合したツールは「経営改善プラットフォーム」になる
- 競合との完全な差別化ポイント（口コミツールはEXを一切扱わない）
- 既に両方のデータ収集基盤は完成済み。表示レイヤーの追加のみ

### 施策6: 「定期検診リマインド効果」の可視化 — 医院経営KPIとの接続

**現状の機能**: 月次レポートで来院数・売上・自費率は記録できるが、「患者体験改善」と「経営指標」の因果が見えない。

**提案**: 患者満足度と経営指標の関係を時系列で並列表示する。

#### 表示イメージ

```
  患者満足度  4.0  4.1  4.3  4.3  4.5  4.5
  来院患者数  180  185  192  195  205  210
  自費率     18%  19%  20%  22%  23%  24%
             1月  2月   3月  4月   5月  6月
                        ↑
                  改善活動開始
```

#### 因果仮説の提示（データ蓄積後）

- 「患者満足度が0.5pt向上した医院は、6ヶ月後の来院数が平均X%増加」
- 「患者満足度4.0以上の医院は自費率がX%高い傾向」
- 根拠: 満足した患者はリコール（定期検診の再来院）率が高く、自費診療の提案も受け入れやすい

#### なぜ重要か
- 院長の最大の関心事は「結局これで売上が上がるのか？」
- 「患者体験が良い → リコール率向上 → 来院数増 → 売上増」のロジックを数字で見せる
- 口コミに頼らない「経営効果の証明」になる
- 既にMonthlyClinicMetricsに来院数・売上データがあるため追加データ収集不要

### 施策7: 業界ベンチマーク — 「自院の立ち位置」を知る

**前提**: 導入クリニック数が一定規模（30院以上）に達した段階で有効化。

**提案**: 匿名化された全クリニックの平均スコアと自院を比較表示する。

#### 表示イメージ

```
┌─ ベンチマーク比較 ──────────────────────────┐
│                                             │
│  御院の患者満足度: 4.3                       │
│  MIERU利用医院平均: 4.1                      │
│  御院の順位: 上位 28%                        │
│                                             │
│  質問別比較:                                 │
│  待ち時間      御院 3.8  平均 3.6  ▲ 平均以上 │
│  説明の分かりやすさ 御院 4.5  平均 4.2  ▲ 平均以上 │
│  スタッフ対応   御院 4.4  平均 4.3  ≒ 平均並み │
│  清潔さ       御院 4.1  平均 4.5  ▼ 改善余地 │
│                                             │
└──────────────────────────────────────────────┘
```

#### なぜ重要か
- 「4.3って高いの？低いの？」が院長の最初の疑問
- 比較対象がないと自院のスコアの意味が分からない
- ベンチマーク機能はSaaS製品のネットワーク効果を生む（利用者が増えるほど価値が上がる）
- 競合他社への乗り換え障壁（Lock-in）にもなる

### Before（転換前）
> 「コンプライアンス準拠の口コミ獲得ツール」
> 比較対象: 口コミPLUS、Hero Innovation、Gyro-nキキコミ

### After（転換後）
> **「患者体験を数値化し、改善サイクルを回す経営改善プラットフォーム」**
> 比較対象: なし（日本市場で新カテゴリ）

### 訴求メッセージの転換

| 項目 | 旧メッセージ | 新メッセージ |
|---|---|---|
| ヘッドライン | 口コミが増える | **患者が戻ってくる医院になる** |
| KPI | Google口コミ件数 | **患者満足度スコアの改善幅** |
| 効果 | 口コミ★4.8達成 | **リコール率向上・自費率向上** |
| 証明 | 口コミ数の増加グラフ | **満足度推移 × 来院数推移の相関** |
| 安心感 | コンプライアンス準拠 | **口コミ誘導を一切行わない唯一のツール** |

### LP（ランディングページ）のFAQ改訂案

現在のFAQ:
> Q: Google口コミの誘導機能はありますか？
> A: MIERU PXは口コミ誘導機能を搭載していません...

改訂案:
> Q: 口コミを増やすツールとは何が違うのですか？
> A: MIERU PXは口コミツールではありません。患者の満足度を測定し、具体的な改善アクションにつなげる「患者体験改善プラットフォーム」です。満足度が上がれば、結果として口コミや再来院率も自然に向上します。口コミの誘導は一切行わず、Googleポリシーに完全に準拠しています。

## 「口コミツールではない」ことの市場価値

### 1. 競合不在のブルーオーシャン
- 日本の口コミツール市場は月額1-3万円で飽和状態
- 「患者体験改善」市場には競合がいない
- 価格競争に巻き込まれず、価値で勝負できる

### 2. 規制リスクからの完全解放
- Google口コミ関連のポリシー変更に影響されない
- ステマ規制の強化を追い風にできる（「うちは口コミ誘導していません」）
- 医療広告ガイドラインの厳格化にも対応済み

### 3. 長期的な差別化（模倣困難性）
- 口コミツールは機能がコモディティ化しやすい（QR→アンケート→誘導、のワンパターン）
- 「改善サイクル」「PX×EX相関」「ベンチマーク」は**データの蓄積自体が参入障壁**になる
- 利用期間が長いほど改善の軌跡が蓄積され、乗り換えコストが高くなる

### 4. より大きな市場への展開可能性
- 口コミツール → 歯科・クリニック限定（ニッチ）
- 患者体験改善プラットフォーム → 病院・介護施設・美容クリニック・動物病院等へ拡大可能
- PX＋EXの統合 → 医療以外（サービス業全般）への応用も将来的に視野
