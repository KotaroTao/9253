# 歯科AI研修事業 改善提案書（10億円ロードマップ版）

## エグゼクティブサマリー

**目標**: Year 1 売上1億円以上 / Year 3 売上10億円以上

この目標は、研修単体の延長では到達不可能。以下の**4つの事業ドメイン**を同時に立ち上げ、相互に強化し合うエコシステムを構築する。

```
┌─────────────────────────────────────────────────────────────┐
│                  歯科DXプラットフォーム事業                    │
│                                                             │
│  ① 研修事業        ② D-Code SaaS     ③ Store      ④ 法人    │
│  （入口）          （基盤）           （拡張）     （拡大）   │
│                                                             │
│  対面研修  ──→  D-Codeライセンス ──→ アプリ販売 ──→ チェーン  │
│  オンライン     API従量課金         手数料収入    一括導入    │
│  フランチャイズ  テンプレート月額     開発者経済圏  コンサル   │
│  法人研修                                         OEM       │
└─────────────────────────────────────────────────────────────┘
```

## 現行モデルの限界

| 指標 | 現行モデル（前回版） | 必要水準 | ギャップ |
|---|---|---|---|
| Year 1 売上 | 2,969万円 | **1億円** | **3.4倍不足** |
| Year 3 売上 | 1.75億円 | **10億円** | **5.7倍不足** |
| 研修だけで1億に必要な人数 | 72名/年 | **303名/年（25名/月）** | 講師1名では物理的に不可能 |

**結論**: 研修の単価・頻度を上げるだけでは不可能。**4つの事業ドメインの同時展開**と**レバレッジの効く収益モデル**が必須。

### 改善1: 研修を「4階建て」に再設計し、単価×客数を最大化

#### 現行の問題
- 1プラン（40万円/4日間）のみ → 客層が限定される
- 講師1名 → スケールの天井が低い

#### 改善案：4階建て研修ラインナップ

```
┌──────────────────────────────────────────────────────────────────────┐
│ 4F  マスタープログラム（年間契約）                                     │
│     300万円/法人  年間12回訪問+Slack常駐サポート+D-Code導入支援        │
│     ターゲット: 5院以上の歯科チェーン                                  │
│     → 年間10法人 = 3,000万円                                         │
├──────────────────────────────────────────────────────────────────────┤
│ 3F  プレミアム研修（対面4日間）                                       │
│     40万円/人（助成金75%で実質10万円）                                │
│     ターゲット: 院長・経営者                                          │
│     → 月8名 × 12ヶ月 = 3,840万円                                    │
├──────────────────────────────────────────────────────────────────────┤
│ 2F  オンライン集中（2日間ライブ）                                     │
│     15万円/人（助成金で実質3.75万円）                                 │
│     ターゲット: 地方の歯科関係者、スタッフ                            │
│     → 月20名 × 12ヶ月 = 3,600万円                                   │
├──────────────────────────────────────────────────────────────────────┤
│ 1F  セルフラーニング（録画+課題）                                     │
│     月額9,800円 or 年額98,000円                                      │
│     ターゲット: まず試したい層                                       │
│     → 月間50名維持 × 9,800円 = 588万円                               │
└──────────────────────────────────────────────────────────────────────┘
```

#### 4F「マスタープログラム」の詳細（最重要の新規商品）

歯科チェーン（5院以上）向けの**年間DXパートナー契約**：

| 内容 | 詳細 |
|---|---|
| 初期研修 | 幹部向け2日間集中（オンサイト） |
| 月次訪問 | 月1回×3時間、現場課題のヒアリング→その場でアプリ改修 |
| Slack常駐 | 専用チャンネルで質問対応（24h以内回答） |
| D-Code導入 | レセコン連携設定、カスタムアプリ3本まで開発込み |
| 定例レポート | 月次DX推進レポート提出 |

**なぜ300万円が通るか**:
- 5院チェーンの月間売上は平均1.5億円。DXで1%改善すれば年間1,800万円の効果
- 300万円は売上の0.2%。ROIは6倍以上
- 助成金適用で実質75万円（IT導入補助金併用の場合）

### 改善2: D-Codeを「研修修了者向けAPI」から「全歯科医院向けSaaS」に拡張

#### 現行の問題
D-Codeが「開発者のためのAPI」に留まると、市場が研修修了者に限定される（数百人規模）。

#### 改善案：D-Codeの3層展開

```
Layer 3: D-Code API（開発者向け）
  月額3万円 + API従量課金
  ターゲット: 研修修了者、歯科系エンジニア
  市場規模: 小（数百名）

Layer 2: D-Code Apps（医院向けSaaS）
  月額1〜5万円/院
  開発者が作ったアプリを医院にSaaSとして提供
  ターゲット: 一般歯科医院（68,000院）
  市場規模: 大

Layer 1: D-Code Templates（ノーコード）← ★新規
  月額9,800円/院
  設定だけで使えるテンプレートアプリ（予約管理、患者説明等）
  ターゲット: IT苦手な歯科医院
  市場規模: 最大
```

**Layer 1「D-Code Templates」が10億への鍵**:
- 日本の歯科医院68,000院のうち、1%でも680院
- 680院 × 月額3万円（平均） = 月間2,040万円 = **年間2.45億円**
- これだけでYear 3売上の約25%を占める

### 改善3: フランチャイズ型講師育成 — 研修の物理的限界を突破

#### 現行の問題
田尾氏1人では月4回×4名 = 月16名が限界。年間192名で売上6,336万円が天井。

#### 改善案：認定講師フランチャイズ制度

```
Year 1: 田尾氏が直接研修しながら、講師候補3名を育成
         └── 研修修了者の中から適性のある歯科医師をスカウト

Year 2: 認定講師4名体制（田尾氏+3名）
         └── 各講師が月2回開催 = 月8回 × 4名 = 月32名
         └── 講師報酬: 売上の40%（12万円/人）
         └── 本部利益: 売上の60%（20万円/人）

Year 3: 認定講師8名体制（全国5都市）
         └── 月16回 × 4名 = 月64名
         └── 年間768名 × 33万円 = 2.53億円（うち本部取り分60% = 1.52億円）
```

#### 講師ロイヤリティモデル

| 項目 | 本部（田尾氏） | 認定講師 |
|---|---|---|
| 研修売上配分 | 60% | 40% |
| D-Code契約の紹介料 | 90% | 10%（初年度のみ） |
| カリキュラム・テンプレート | 提供 | 利用権 |
| ブランド使用 | 所有 | ライセンス |
| 集客 | 本部が一括 | 自主集客分は配分70% |

### 改善4: 法人チャネルの大型化 — レセコン会社・ディーラーとのOEM

#### 現行の問題
個別医院への営業は効率が悪い。68,000院に1院ずつアプローチするのは非現実的。

#### 改善案：3つの法人チャネル

#### 4-1. レセコン会社とのOEM/バンドル

```
提案: D-Code TemplatesをレセコンのオプションサービスとしてOEM提供

レセコン会社のメリット:
├── 月額課金の追加収入（ARPU向上）
├── 解約率の低下（付加価値増）
├── 「AI対応レセコン」としてのブランディング
└── 開発コストゼロ（D-Code側が開発・保守）

収益分配: レセコン会社 30% / D-Code 70%

試算:
├── レセコン大手1社のシェア: 約10,000院
├── オプション申込率: 5%（500院）
├── 月額2万円（レセコン経由割引価格）
├── 年間売上: 500院 × 2万円 × 12ヶ月 = 1.2億円
└── D-Code取り分（70%）: 8,400万円
```

#### 4-2. 歯科ディーラー経由の販売

```
日本の主要歯科ディーラー:
├── モリタ（シェア25%）
├── ヨシダ（シェア15%）
├── 歯愛メディカル
└── その他地域ディーラー

提案: ディーラーの営業担当がD-Codeを「DX支援サービス」として販売
├── ディーラー手数料: 月額の20%（永続）
├── ディーラーのメリット: 既存顧客への新規提案材料、関係強化
└── 年間1,000院獲得が現実的目標
```

#### 4-3. 歯科医師会・学会との連携

```
提案: 都道府県歯科医師会の「DX研修」として公式採用
├── 歯科医師会が会員向けに研修を斡旋
├── 受講料の一部を医師会が負担（生涯学習ポイント付与）
├── 年間20都道府県 × 各20名 = 400名
└── 400名 × 15万円（団体割引） = 6,000万円
```

### 改善5: 歯科→医科→介護への業種展開（Year 2後半〜）

#### 現行の問題
歯科だけでは市場上限がある（68,000院 × 月額3万円 = 年間245億円の理論最大値だが、現実的なシェアは1%未満）。

#### 改善案：隣接市場への段階的展開

```
Year 1-2: 歯科特化（68,000院）
           └── ここで型を作る

Year 2後半: 医科クリニック展開（102,000院）
           └── D-Code → M-Code（Medical Code）
           └── レセコンは異なるが、テンプレート構造は流用可能

Year 3: 介護・薬局展開
           └── 介護施設: 約40,000施設
           └── 薬局: 約60,000店
           └── 合計TAM: 270,000施設

市場拡大効果:
├── 歯科のみ: TAM 245億円
├── 医科追加: TAM 612億円（+367億円）
└── 全領域:  TAM 972億円（+727億円）
```

### 改善6: Store手数料モデルの再設計 — 「歯科AppStore」

#### 改善案

```
旧モデル: 開発者が作ったアプリの手数料30%
問題: 開発者が少ないと品質・量ともに不足

新モデル: 3つの収益源を統合

1. 自社開発テンプレートアプリ（利益率90%以上）
   └── 月額5,000〜30,000円 × 数百院
   └── 自社でClaude Codeを使って高速開発

2. パートナー開発アプリ（手数料30%）
   └── 研修修了者・外部開発者のアプリ
   └── 品質が担保できたもののみ掲載

3. エンタープライズカスタム（1件100〜500万円）
   └── 大手チェーン向けのカスタム開発
   └── テンプレートをベースに個別要件を実装
```

## 最終提言

### Year 1で1億を達成する最短ルート

1. **マスタープログラム（300万円法人契約）を最優先で5件取る** — これだけで1,500万円。大手チェーンへのトップ営業が最重要タスク
2. **歯科医師会を5都道府県開拓する** — 1都道府県あたり20名×15万円 = 300万円。5つで1,500万円。医師会ルートは信用力が高く、受講者も集まりやすい
3. **オンライン研修を月15名ペースに乗せる** — 地方の歯科医師にリーチ。対面研修の1/3の価格で参入障壁を下げる

この3つで4,200万円。残り5,800万円は対面研修（2,970万円）+ D-Code + コンサル + スポンサーで積み上げ可能。

### Year 3で10億を達成するための分水嶺

**D-Code Templatesが1,000院を超えるかどうか**。これが事業の成否を分ける。

- 1,000院 × 月額3万円 = MRR 3,000万円 = ARR 3.6億円
- SaaS事業としてのトラクションが証明されれば、VCからの調達も可能
- 調達資金で医科展開を加速 → TAM 3倍

**研修は入口であり、D-Codeが本体**。この認識を初日から持つことが最重要。
